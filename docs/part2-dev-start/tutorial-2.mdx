# 案例二：区块链隐私交易

## 场景

在传统区块链上，所有的交易金额、余额都是公开的。假设 Alice 想向 Bob 转账，但她不想让别人知道自己有多少钱、转了多少。

使用 FHE 技术，可以在区块链上实现**机密 ERC-20 Token**，让交易金额和余额保持加密状态。

## 官方示例

OpenZeppelin 提供了基于 fhEVM 的机密代币标准 **ERC-7984**。

## ERC-7984 简介

ERC-7984 是 OpenZeppelin 开发的机密代币标准，类似于 ERC-20，但所有余额和转账金额都表示为密文句柄，确保不会有数据泄露给公众。

## 官方机密代币示例

来源：[OpenZeppelin ERC-7984](https://docs.openzeppelin.com/confidential-contracts/token)

### 铸造/销毁合约

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.24;

import {FHE, externalEuint64} from "@fhevm/solidity/lib/FHE.sol";
import {Ownable} from "@openzeppelin/contracts/access/Ownable.sol";
import {ERC7984} from "@openzeppelin/confidential-contracts/token/ERC7984/ERC7984.sol";

contract ERC7984MintableBurnable is ERC7984, Ownable {
    constructor(
        address owner,
        string memory name,
        string memory symbol,
        string memory uri
    ) ERC7984(name, symbol, uri) Ownable(owner) {}

    function mint(address to, externalEuint64 amount, bytes memory inputProof) public onlyOwner {
        _mint(to, FHE.fromExternal(amount, inputProof));
    }

    function burn(address from, externalEuint64 amount, bytes memory inputProof) public onlyOwner {
        _burn(from, FHE.fromExternal(amount, inputProof));
    }
}
```

### 操作员（Operator）设置

```typescript
const alice: Wallet;
const expirationTimestamp = Math.round(Date.now()) + 60 * 60 * 24; // Now + 24 hours

await tokenContract.connect(alice).setOperator(bob, expirationTimestamp);
```

## 机密转账原理

```
传统 ERC-20 交易：
Alice 转账 100 USDC 给 Bob
→ 所有人可见：Alice 地址、Bob 地址、转账金额

机密 ERC-7984 交易：
Alice 转账 E(100) USDC 给 Bob（E 表示加密）
→ 只有 Alice 和 Bob 知道实际金额
→ 验证者可以验证交易有效性，但不知道具体金额
```

## 与 ZK 的对比

| 特性 | FHE 方案 | ZK 方案（如 Tornado Cash） |
|------|----------|---------------------------|
| 隐藏内容 | 交易金额、余额 | 交易关联性（谁转给谁） |
| 透明性 | 交易本身公开，金额加密 | 交易公开，无金额信息 |
| 计算开销 | 高（密文运算） | 中等（证明生成） |
| 适用场景 | 保密金额的转账 | 匿名混币 |

## 关键点总结

1. **金额加密**：交易金额和余额在整个过程中保持加密
2. **可验证性**：验证者可以在不解密的情况下验证交易有效性
3. **选择性披露**：只有拥有私钥的用户才能解密查看实际金额
4. **合规性**：结合 ZK 可以实现监管可追溯性
