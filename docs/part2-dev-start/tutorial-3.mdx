# 案例三：构建你的第一个 FHE 应用

## 环境准备

本案例使用 **TFHE-rs**（Rust 实现），由 Zama 提供。

### 系统要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Linux (Ubuntu 20.04+), macOS, Windows (WSL2) |
| 内存 | 最少 8GB（推荐 16GB+） |
| Rust | 1.60+ |

### 安装步骤

```bash
# 1. 安装 Rust
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source ~/.cargo/env

# 2. 创建项目
cargo new my-fhe-app
cd my-fhe-app

# 3. 添加 TFHE-rs 依赖
cargo add tfhe --features boolean,shortint,integer

# 4. 编译
cargo build --release
```

### 注意事项

:::warning 首次编译时间长
TFHE-rs 编译时间较长，建议耐心等待或使用预编译的 Docker 镜像。
:::

:::warning 密钥安全
- 私钥（client_key）绝不能发送到服务器
- 只发送公钥（public_key）和计算钥（server_key）给服务器
:::

## 项目结构建议

```
my-fhe-app/
├── src/

│   └── common/          # 共享类型和工具
```

## 使用 TFHE-rs 构建应用

### 1. 创建 Rust 项目

```bash
cargo new my-fhe-app
cd my-fhe-app
```

### 2. 添加依赖

```toml
[dependencies]
tfhe = { version = "0.6", features = ["integer", "borsh"] }
```

### 3. 完整示例：隐私投票系统

```rust
use tfhe::prelude::*;
use tfhe::{generate_keys, set_server_key, ConfigBuilder};
use tfhe::integer::{FheUint8, ServerUint8};

fn main() {
    // === 服务器端（数据收集方）===
    let config = ConfigBuilder::default();
    let (client_key, server_key) = generate_keys(config);
    set_server_key(server_key);

    // === 客户端（投票者）===
    // 假设三个选项：0=反对, 1=赞成, 2=弃权
    let my_vote: u8 = 1; // 我投 "赞成"

    // 加密投票
    let encrypted_vote = FheUint8::encrypt(my_vote, &client_key);

    // === 服务器端：汇总票数 ===
    // 模拟 100 个投票者（这里简化处理）
    let mut total_yes = encrypted_vote.clone();
    let mut total_no = FheUint8::encrypt(0, &client_key);
    let mut total_abstain = FheUint8::encrypt(0, &client_key);

    // 批量处理（实际中从网络接收密文）
    // 统计赞成票
    let is_yes = encrypted_vote.clone().eq(&FheUint8::encrypt(1, &client_key));
    // ... 实际需要根据具体值做条件计数

    // === 服务器返回结果 ===
    let result_ct = total_yes;

    // === 客户端解密：查看结果 ===
    let result: u8 = result_ct.decrypt(&client_key);
    println!("赞成票数: {}", result);
}
```

## 常见问题

| 问题 | 解决方案 |
|------|----------|
| 编译太慢 | 使用 `cargo build --release` |
| 内存不足 | 减小 poly_modulus_degree |
| 精度不够 | 选择更大的参数或 CKKS |
