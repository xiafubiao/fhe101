# 什么是同态加密？

## 从"数据外包"说起

想象一下，你有一份敏感的医疗报告，想让云端的人工智能帮你分析，但你又不想让云端看到报告的内容，怎么办？

传统加密：数据加密后存放在云端，但云端**无法对加密数据做任何运算**。要想分析？必须先解密。

**同态加密**：一种特殊的加密技术，让云端可以直接对**密文**进行运算，运算后的结果解密后恰好等于直接在原文上运算的结果。

## 同态加密家族

同态加密不是一夜之间诞生的，它经历了漫长的技术演进：

| 类型 | 简称 | 说明 | 代表方案 |
|------|------|------|----------|
| 部分同态加密 | PHE | 只支持加法或乘法中的一种 | RSA, Paillier |
| 有点同态加密 | SWHE | 支持有限次数的加法和乘法 | BGN |
| **全同态加密** | **FHE** | 支持任意次数的加法和乘法 | BFV, CKKS, TFHE |

### 半同态的局限

早期的加密方案只能支持单一运算：

**RSA（同态乘法）**：
```
E(x) = x^e mod n
E(y) = y^e mod n
E(x) × E(y) = (xy)^e mod n = E(xy)  ✓
```

**Paillier（同态加法）**：
```
E(x) = g^x × r^n mod n²
E(y) = g^y × s^n mod n²
E(x) × E(y) = g^(x+y) × (rs)^n mod n² = E(x+y)  ✓
```

但无法同时支持加法和乘法，这限制了实际应用。

## FHE 的核心公式

全同态加密支持在密文上进行任意次数的加法和乘法运算：

```
E(x + y) = E(x) ⊕ E(y)           (加法同态)
E(x × y) = E(x) ⊗ E(y)           (乘法同态)
E(x - y) = E(x) ⊕ E⁻¹ ⊗ E(y)    (减法同态)
```

> **直观理解**
> 这意味着：
> - **直接在密文上运算**的结果
> - **解密后**恰好等于**在原文上运算**的结果
> - 运算方全程不知道原始数据是什么

## 用生活例子理解

> 把你的数据想象成放入一个**特殊的锁盒子**。你有一把私钥可以打开它，但云端没有。
>
> 你把锁盒子交给云端服务器。云端可以在不打开盒子的情况下，往里面加东西、乘以某个数。
>
> 等云端运算完毕，把盒子还给你。你用私钥打开，发现里面的数据正是你想要的运算结果——而云端从头到尾都不知道里面装的是什么。

## FHE 的里程碑

| 年份 | 事件 | 意义 |
|------|------|------|
| 1978 | Rivest 等提出同态加密概念 | 理论萌芽 |
| 1999 | Paillier 算法提出 | 首个实用加法同态 |
| **2009** | **Gentry 基于理想格实现 FHE** | **突破性进展** |
| 2011 | BV 算法 | 基于标准 LWE |
| 2012 | BGV 算法 | 层级式 FHE |
| 2012 | BFV 算法 | 整数批处理 |
| 2013 | GSW 算法 | 近似 DNFS 假设 |
| 2017 | CKKS | 浮点近似计算，ML 应用 |
| TFHE | 2016-至今 | 布尔电路快速运算 |

## 小结

FHE（Fully Homomorphic Encryption，全同态加密）让数据在整个处理过程中都保持加密状态，解决了"数据可用不可见"的终极隐私问题。它是密码学皇冠上的明珠，也是隐私计算时代的重要基础设施。
