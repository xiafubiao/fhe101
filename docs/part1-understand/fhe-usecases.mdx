# FHE 能做什么？不能做什么？

## FHE 的典型应用场景

### 1. 云端隐私计算
将数据加密后发送到云端，云端在密文上执行分析或机器学习推理，数据永远以密文形式存在。

**案例**：医疗影像分析
- 患者加密自己的 CT 影像
- 云端 AI 在密文上运行肿瘤检测模型
- 返回加密结果，患者用私钥解密查看

### 2. 经典案例：百万富翁问题（Millionaires' Problem）

两个百万富翁想知道谁更有钱，但都不想让对方知道自己有多少钱。

**传统方案**：需要引入可信第三方

**FHE 方案**：
```
Alice 的资产：x 百万美元
Bob 的资产：y 百万美元

1. Alice 用 FHE 公钥加密 x，发送给 Bob
2. Bob 收到后，构造密文比较 x 和 y
3. Bob 返回比较结果（仍为密文）
4. Alice 用私钥解密，知道 x > y 或 x < y
5. 双方只知道比较结果，不知道对方具体资产
```

> **延伸应用**
> 这个模式可以扩展到：
> - 隐私竞价：竞标者不知道他人的出价
> - 薪资比较：员工不知道自己比别人低多少
> - 信用评分：银行不知道你的具体分数

### 3. 隐私机器学习
模型参数和输入数据都可以保持加密状态。

**场景**：
- 隐私保护的推荐系统
- 加密数据上的模型训练
- 联邦学习中的梯度保护

**案例：医疗诊断模型**
```
1. 医院 A 拥有加密的患者数据
2. AI 公司提供加密的模型参数
3. 云端在密文上执行推理
4. 返回加密诊断结果
5. 医院 A 解密得到诊断（医院不知道模型细节，AI 公司不知道患者数据）
```

### 4. 区块链与 Web3
结合 FHE 可以实现：
- **私密交易**：交易金额对第三方不可见
- **去中心化身份认证**：验证身份而不泄露信息
- **链上隐私投票**：投票内容保密

### 5. 数据安全共享
在多个组织间共享数据时：
- 各方数据始终加密
- 联合分析只返回聚合结果
- 合规性更强（GDPR 等）

**案例：联合风控**
```
银行 A + 银行 B + 银行 C → 计算行业平均不良率

- 每家银行只看到自己和其他家联合计算的结果
- 任何单一银行无法获取其他家的明细数据
- 监管部门可以验证结果的正确性（结合 ZK）
```

## FHE 当前局限性

### 性能开销
| 操作类型 | 开销倍数（vs 明文） |
|----------|---------------------|
| 加法 | ~1000x |
| 乘法 | ~10000x |
| Bootstrapping | ~100000x |

**建议**：实际应用中，尽量减少密文运算次数。

### 功能限制
- **非线性运算困难**：比较、除法等需要特殊技巧
- **浮点精度有限**：CKKS 方案是近似计算
- **噪声累积**：运算次数受噪声预算限制

### 使用复杂度
- 参数选择需要专业知识
- 密钥管理复杂
- 调试困难（密文无法直接查看）

## 不能用 FHE 的场景

| 场景 | 替代方案 |
|------|----------|
| 需要实时高频交易 | TEE |
| 多方需要互相验证身份 | ZK |
| 数据量极大但预算有限 | MPC + 差分隐私 |
| 需要任意非线性函数 | 混合方案（FHE + 近似算法） |

## 技术发展趋势

FHE 正在快速成熟：

| 方向 | 进展 |
|------|------|
| **硬件加速** | FHE 专用芯片研发中（Intel, DARPA） |
| **算法优化** | 新 bootstrapping 算法不断降低开销 |
| **工具完善** | 开发者体验显著改善 |
| **标准化** | HomomorphicEncryption.org 推进标准化 |

未来 5-10 年，FHE 有望在更多场景中实用化。
