# FHE Operation Handling

## Non-Linear Operation Handling

FHE has weak support for non-linear operations (comparisons, division), requiring special techniques.

### Comparison Operations

Ciphertext comparison is a core difficulty in FHE applications because:

- Addition, multiplication: Naturally supported
- Comparison (greater than, less than, equal): Need to be converted to arithmetic operations

#### Sign Bit Method

A common method is to compare the difference of two numbers:

```python
# Compare if x > y
# Equivalent to judging the sign of (x - y)

diff = x - y
# If diff > 0, then x > y
# If diff < 0, then x < y
# If diff = 0, then x = y
```

The problem is: FHE ciphertext comparison cannot directly get a boolean result, requiring special encoding.

#### 0-Encoding and 1-Encoding Methods

This is a two-round solution proposed by Lin et al., used to solve Yao's Millionaires' Problem.

**Core Idea**:
For binary string `s = s_n s_{n-1} ... s_1`

```
0-encoding (S_s^0):
  Replace the first 0 in s with 1, keep other bits unchanged
  S_s^0 = { s_n ... s_{i+1} 1 s_i = 0, 1 ≤ i ≤ n }

1-encoding (S_s^1):
  Replace the first 1 in s with 1, keep other bits unchanged
  S_s^1 = { s_n ... s_{i+1} 1 s_i = 1, 1 ≤ i ≤ n }
```

**Comparison Principle**:
- x > y iff x's 1-encoding and y's 0-encoding have common elements

```python
# Simplified example (4-bit integers)
x = 5 = 0101 (binary)
y = 3 = 0011 (binary)

# x > y condition encoding
x's 1-encoding: {1101, 0111, 0100, 0101}
y's 0-encoding: {1011, 0010, 0001, 0000}

# Check for intersection
# 0101 ∈ x's 1-encoding
# 0101 ∈ y's 0-encoding? Yes (corresponds to i=4)
# Intersection exists → x > y holds ✓
```

### Division Operations

Division in FHE needs to be implemented using **reciprocal approximation**:

```python
# Calculate x / y
# 1. Approximately calculate reciprocal of y: y^(-1) ≈ polynomial approximation
reciprocal = approximate_reciprocal(y, degree=3)
# 2. Ciphertext multiplication
result = x * reciprocal
# 3. Note: This is approximate result, limited precision
```

Common approximation methods:
- Newton's iteration method
- Taylor series expansion
- Chebyshev polynomials

## Programmable FHE

Modern FHE supports user-defined operations without pre-built specific functions:

```rust
// TFHE-rs: Define custom operations
let custom_gelu = |x: &FheFloat| {
    // Approximate implementation of GELU activation function
    0.5 * x * (1.0 + tanh(0.797884 * (x + 0.044715 * x * x * x)))
};
```

```python
# Concrete (Zama): FHE-friendly machine learning compiler
# Automatically converts PyTorch models to approximate FHE-executable versions
from concrete import fhe
import torch.nn as nn

# User just adds @fhe decorator
@fhe.compile
def my_model(x):
    return torch.sigmoid(x @ torch.tensor([[0.5]]))
```
